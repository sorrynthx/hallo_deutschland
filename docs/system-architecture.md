# System Architecture (시스템 아키텍처)

이 문서는 학습 애플리케이션의 데이터 흐름(Data Flow)과 시퀀스 흐름(Sequence Flow)을 한눈에 파악할 수 있도록 텍스트로 시각화한 문서입니다. 신입 개발자가 애플리케이션의 전반적인 구조를 이해하는 데 도움을 주기 위해 작성되었습니다.

## 1. Data Flow (데이터 플로우)

애플리케이션은 데이터베이스 대신 정적 JSON 파일을 사용하여 콘텐츠(어휘, 문법 등)를 제공합니다. 사용자의 학습 진행도는 로컬 스토리지(`localStorage`)에 저장됩니다.

```text
[ 데이터 흐름도 ]

+-------------------+       +--------------------+       +-------------------+
|  Static JSON      |       |  Next.js App       |       |  Local Storage    |
|  (public/data/)   | ----> |  (Client-side)     | <---> |  (User Browser)   |
|  - vocab.json     | Fetch |  - page.tsx        | Save/ |  - progress data  |
|  - grammar.json   |       |  - Quiz Components | Load  |                   |
+-------------------+       +--------------------+       +-------------------+
                                     |
                                     v
                            +--------------------+
                            |  UI Components     |
                            |  - QuizCard        |
                            |  - ProgressBar     |
                            |  - FeedbackBanner  |
                            +--------------------+
```

### 주요 컴포넌트 데이터 전달 흐름
1. `app/learn/[locationId]/[typeId]/page.tsx` 라우트에서 URL 파라미터를 기반으로 적절한 JSON 데이터를 `fetch` 합니다.
2. 불러온 데이터는 학습 타입(vocab, grammar, expressions 등)에 맞는 퀴즈 컴포넌트로 `props`를 통해 전달됩니다.
3. 퀴즈 컴포넌트 내부에서는 `useQuizLogic` 훅을 사용하여 현재 문제 인덱스(index)와 점수(score) 상태를 관리합니다.
4. 학습이 완료되면 `useProgress` 훅을 통해 로컬 스토리지에 사용자의 진행도를 업데이트합니다.

---

## 2. Sequence Flow (시퀀스 플로우)

사용자가 퀴즈 페이지에 접속하여 학습을 완료하기까지의 일련의 과정입니다.

```text
[ 퀴즈 진행 시퀀스도 ]

User             Page (page.tsx)     Quiz Component      useQuizLogic      Local Storage
 |                      |                   |                  |                 |
 | 1. 접속 (URL 이동)   |                   |                  |                 |
 |--------------------->|                   |                  |                 |
 |                      | 2. JSON 데이터    |                  |                 |
 |                      |    Fetch          |                  |                 |
 |                      |------------------>|                  |                 |
 |                      |                   | 3. 초기 상태설정 |                 |
 |                      |                   |----------------->|                 |
 |                      |                   |                  |                 |
 |                      | 4. 첫 문제 렌더링 |                  |                 |
 |<---------------------|-------------------|                  |                 |
 |                      |                   |                  |                 |
 | 5. 정답/오답 선택    |                   |                  |                 |
 |--------------------->|                   |                  |                 |
 |                      |                   | 6. 상태 업데이트 |                 |
 |                      |                   |----------------->|                 |
 |                      | 7. 다음 문제 렌더 |                  |                 |
 |<---------------------|-------------------|                  |                 |
 |                      |                   |                  |                 |
 ... (반복)             |                   |                  |                 |
 |                      |                   |                  |                 |
 | 8. 마지막 문제 완료  |                   |                  |                 |
 |--------------------->|                   |                  |                 |
 |                      |                   | 9. 학습 결과     |                 |
 |                      |                   |----------------->|                 |
 |                      |                   |                  | 10. 진행도 저장 |
 |                      |                   |                  |---------------->|
 |                      | 11. 결과화면 표시 |                  |                 |
 |<---------------------|-------------------|                  |                 |
```

### 시퀀스 상세 설명
1. **접속 및 데이터 로드**: 사용자가 특정 학습 페이지에 접근하면 페이지 컴포넌트는 필요한 JSON 파일을 비동기적으로 가져옵니다.
2. **초기화**: 데이터를 기반으로 퀴즈 컴포넌트가 마운트되며, `useQuizLogic` 훅이 인덱스, 점수 등의 초기 상태를 설정합니다.
3. **사용자 인터랙션**: 사용자가 정답 또는 오답 버튼을 클릭합니다 (예: VocabQuiz의 '알아요', '몰라요' 버튼).
4. **상태 업데이트 및 진행**: 컴포넌트는 내부적으로 인덱스를 증가시키고 새로운 문제를 렌더링합니다.
5. **완료 및 저장**: 모든 문제를 풀면 학습 완료 상태(done)가 되며, `CompleteScreen`이 표시됩니다. 이때 `useProgress` 훅이 호출되어 로컬 스토리지에 학습 이력을 저장합니다.
